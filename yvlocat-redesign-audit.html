Layout Audit · YvloCat Website

What's Wrong &
How to Fix It

A complete visual layout diagnosis — mobile and desktop — focused on structure, sizing, spacing, and framing. No aesthetics. Pure layout.

Foundation

What a Professional Layout Does

Before diagnosing problems, here's what both versions of the site need to achieve.

📐

Every element earns its space

No container should exist unless it contains meaningful content. Empty padding is a symptom, not a style choice.

📱

Mobile is the primary canvas

Design for the smallest screen first. Desktop is an expansion of mobile, not a shrunk-down desktop layout.

↔️

Columns must earn their place

Two columns on mobile almost always fail. Use them only when both columns have equally complex, equal-weight content.

🧱

Consistent grid & max-width

Content should never touch screen edges (min 16px gutter). On desktop, text should never span the full width of a wide viewport.

Problem Areas

Issue-by-Issue Breakdown

Ordered by severity. Each issue explains what's wrong on both screens and what to do instead.

1
Two-column layout inside the booking section on mobile High
❌ Problem

The "Craft Your Reading" form and "Your Quote" price card sit side by side at roughly 50/50 width on mobile. Each column is ~160px wide. Text wraps badly, inputs are barely tappable, the price summary is unreadable. This is the single biggest usability failure on the site.

✅ Fix

Stack them vertically, full width, on mobile. "Craft Your Reading" goes on top. "Your Quote" goes below it. On desktop (≥768px) only, use a two-column layout with the form taking 55% and the quote taking 45%.

/* WRONG — always two columns */
.booking-grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
}

/* RIGHT — stack on mobile, side-by-side on desktop */
.booking-grid {
  display: grid;
  grid-template-columns: 1fr;    /* mobile default */
  gap: 24px;
}

@media (min-width: 768px) {
  .booking-grid {
    grid-template-columns: 55fr 45fr;
  }
}
2
Mascot container — oversized empty card around a small image High
❌ Problem

The witch cat mascot sits inside a large rounded rectangle with ~80px of vertical padding above and below it. On mobile this consumes roughly 30% of screen height for a single small image. On desktop the card gets even wider but the cat stays small, making the dead space worse. The container signals "something important is here" but delivers nothing.

✅ Fix

Remove the standalone card. Integrate the mascot inline — either as a decorative element between the hero headline and CTA buttons, or floated beside the value props list. Give it a natural size (80–120px). Let the page breathe around it naturally, not through inflated padding.

3
Value props card — excessive vertical padding, text too centered High
❌ Problem

The three numbered items (01 Gentle Clarity, 02 Tailored Depth, 03 Personal Care) are stacked in a single tall card. Each row has generous padding that makes it feel like three separate cards squeezed together. On desktop the card stretches to full container width but the text stays short and centered, leaving wide dead zones on both sides.

✅ Fix

On mobile: reduce vertical padding between items to 16px. Use a clear divider line between rows but tight spacing. On desktop: switch to a 3-column horizontal layout — each value prop in its own column, icon above, number and title below. This uses the wide viewport productively.

/* Value props: stacked on mobile, 3-col on desktop */
.value-props {
  display: grid;
  grid-template-columns: 1fr;
  gap: 0;
}
.value-prop-item {
  padding: 16px 20px;    /* was probably 28px+ */
  border-bottom: 1px solid var(--border);
}

@media (min-width: 768px) {
  .value-props {
    grid-template-columns: repeat(3, 1fr);
    gap: 1px;                 /* grid gap as divider */
    background: var(--border);  /* gap shows border color */
    border-radius: 12px;
    overflow: hidden;
  }
  .value-prop-item {
    background: white;
    padding: 28px 24px;
    border-bottom: none;
  }
}
4
Navigation bar — protruding Book button breaks the bar's frame Medium
❌ Problem

The purple circle "Book" button rises above the nav bar's top edge. The nav bar has a defined height and rounded corners, but the button pokes outside that boundary. This looks unintentional on mobile and especially broken on desktop where the bar floats at the bottom with visible geometry. The eye sees the broken frame before it sees the button's purpose.

✅ Fix

Option A: Increase the nav bar height (e.g. 72px) so the taller button sits within it with clearance above. Option B: Remove the raised treatment entirely and style the Book tab as a filled pill button within the bar. Option C: If you keep the elevated button, it needs a matching notch/cutout in the bar so the bar clearly accommodates it — not fights it.

/* Option A — raise the bar to contain the button */
.bottom-nav {
  height: 72px;              /* was likely 56px */
  padding-bottom: env(safe-area-inset-bottom);
  align-items: center;
}
.bottom-nav .book-btn {
  width: 52px;
  height: 52px;
  margin-top: 0;            /* remove negative margin lift */
}
⚠️ Also note

On desktop, a bottom floating nav bar is unusual. Consider a conventional top navigation bar for desktop viewports (≥1024px). Reserve the bottom nav for mobile only.

5
No max-width constraint on desktop — content stretches too wide Medium
❌ Problem

On wide desktop screens, sections appear to span the full viewport width. Text lines become too long to read comfortably (ideal reading line length is 60–75 characters). Cards stretch to fill wide containers, making them feel thin and spreadsheet-like rather than purposeful and contained.

✅ Fix

Wrap all page content in a centered container with a max-width of 1080px. Within that, constrain prose text further to ~680px. This is the single most impactful desktop fix — it immediately makes the page feel professional and intentional.

.page-container {
  width: 100%;
  max-width: 1080px;
  margin: 0 auto;
  padding: 0 24px;         /* gutters on mobile */
}
@media (min-width: 768px) {
  .page-container {
    padding: 0 48px;       /* wider gutters on desktop */
  }
}
.prose-text {
  max-width: 680px;        /* constrain reading text */
  margin: 0 auto;
}
6
Hero section — headline and CTA not visually anchored Medium
❌ Problem

The hero has a headline, a subtitle, two CTA buttons, and a "Change Region Pricing" link — but they appear to float without clear vertical rhythm. The two CTA buttons are equal weight when one (Get My Reading) should dominate. The region link sits below with unclear hierarchy. On desktop, the hero stays narrow when it could use the width to create a stronger impression.

✅ Fix

Create a clear 3-level hierarchy: (1) headline large + bold, (2) subtitle smaller + muted, (3) primary CTA prominent + full-width on mobile, secondary CTA ghost/outline. "Change Region Pricing" should be a small text link, not a button shape. On desktop, consider a split layout — text left, decorative element right.

/* Button hierarchy */
.btn-primary {
  width: 100%;               /* full width on mobile */
  padding: 14px 24px;
  font-size: 16px;
  font-weight: 600;
}
.btn-secondary {
  width: 100%;
  padding: 12px 24px;       /* slightly smaller */
  font-size: 15px;
  border: 1.5px solid currentColor;
  background: transparent;
}
.link-minor {
  font-size: 13px;
  text-decoration: underline;
  background: none;         /* NOT a button */
  border: none;
}
7
Touch target sizes — inputs and buttons too small on mobile Medium
❌ Problem

When the booking section is squeezed into two columns on mobile, the budget input and interactive elements shrink below Apple/Google's minimum recommended touch target of 44×44px. This makes them hard to tap accurately and feels unfinished.

✅ Fix

All interactive elements (inputs, buttons, nav items) must be at least 44px tall on mobile. Inputs should be 48px tall minimum. This is a baseline accessibility and usability requirement, not a design preference.

input, select, textarea, button {
  min-height: 44px;
}
input[type="number"], input[type="text"] {
  height: 48px;
  padding: 0 16px;
  font-size: 16px;  /* prevents iOS zoom-in on focus */
  width: 100%;
}

Final Check

Professional Layout Checklist

Run through these before publishing. If any item fails, it'll be visible to users.

  • □ No two-column layouts on mobile unless both columns have equal, complex content that genuinely benefits from comparison.
  • □ All interactive elements ≥ 44px tall on mobile. Inputs are 48px. Font-size inside inputs is 16px (prevents iOS auto-zoom).
  • □ Max-width set on all content containers so text doesn't span more than ~80 chars on wide screens.
  • □ Navigation is appropriate per breakpoint — bottom bar on mobile only, top nav on desktop.
  • □ No element floats outside its parent container — the Book button must be fully contained within the nav bar.
  • □ No standalone containers whose only purpose is padding around a small element. The mascot card is the test case here.
  • □ Page content never touches screen edges — minimum 16px gutter on mobile, 48px on desktop.
  • □ Primary CTA is visually dominant over secondary CTA — clear hierarchy between the two buttons.
  • □ Test at 320px width (iPhone SE size) — the smallest common screen. If it breaks here, it's not truly mobile-first.
  • □ Test at 1440px width — the most common desktop resolution. Ensure max-width kicks in and content doesn't stretch uncomfortably.
Priority Order

What to Fix First

If you're doing this in stages, tackle them in this order for the biggest impact per hour of work.

  • 1.Stack the booking section vertically on mobile. This is the biggest usability failure and the most visible problem.
  • 2.Add max-width to content containers on desktop. One CSS rule that makes the whole site feel professional immediately.
  • 3.Remove the mascot standalone card. Integrate it into the hero. Eliminates the dead-space problem entirely.
  • 4.Fix the nav button so it doesn't protrude outside the bar. Raise bar height or remove the lift effect.
  • 5.Switch to a 3-column value props layout on desktop. Converts dead horizontal space into purposeful structure.
  • 6.Set input heights and font sizes to mobile-safe values. Small but important for usability and preventing iOS zoom.
